# Phase v0.4 — Consolidation (Dream Cycle + CLS)

> Consolidation memoire, memory linking, episodique → semantique.
> Le sommeil de hora.

---

## Sous-phases

### v0.4a — Episode Management (1-2 jours)

- [ ] `core/episode.rs` : gestion complete des episodes
- [ ] `add_episode(source, session_id, entity_ids, fact_ids)` : enregistrer un episode
- [ ] `get_episodes(filter)` : recuperer les episodes par source, date, entites
- [ ] `consolidation_count` sur chaque episode : combien de fois consolide
- [ ] Episodes tries par created_at

**Tests :**
- add_episode avec 3 entites et 2 faits : recuperable
- get_episodes par session_id : filtre correct
- get_episodes par date range : correct
- consolidation_count initial = 0

### v0.4b — SHY Downscaling (1 jour)

- [ ] `memory/consolidation.rs` : shy_downscaling()
- [ ] ∀ entity: activation *= downscaling_factor (defaut: 0.78)
- [ ] Idempotence : deux appels consecutifs = deux reductions (c'est voulu, homeostatique)
- [ ] Configurable : factor et quelles entites (toutes par defaut)

**Tests :**
- Apres SHY : toutes les activations reduites de 22%
- Entite avec activation=1.0 → 0.78
- Entite avec activation=-1.0 → -0.78 (meme les negatives)
- Double SHY : 1.0 → 0.78 → 0.608

### v0.4c — Interleaved Replay (2-3 jours)

- [ ] `memory/consolidation.rs` : interleaved_replay()
- [ ] Selection mix : 70% episodes recents + 30% episodes anciens
- [ ] Pour chaque episode selectionne : re-activer les entites mentionnees
- [ ] Re-activation = record_access() → BLL boost
- [ ] Limiter a max_replay_items (defaut: 100) par cycle
- [ ] Selection aleatoire ponderee par age (les recents plus probables)

**Tests :**
- Replay 10 episodes → les entites mentionnees ont une activation plus haute
- Mix 70/30 : verifier la proportion (tolerance ±10%)
- Max replay items respecte
- Episodes sans entites valides : ignores silencieusement

### v0.4d — CLS Transfer (2-3 jours)

- [ ] `memory/consolidation.rs` : cls_transfer()
- [ ] Scanner les episodes avec consolidation_count >= threshold (defaut: 3)
- [ ] Identifier les faits recurrents (meme relation source→target vue 3+ fois)
- [ ] Pour chaque fait recurrent : creer/renforcer un fait semantique
- [ ] Fait semantique : confidence augmentee, pas de source episodique
- [ ] Incrementer consolidation_count des episodes traites

**Tests :**
- Fait vu dans 3 episodes differents → cree un fait semantique
- Fait vu dans 2 episodes → pas encore transfere
- Fait semantique deja existant → confidence renforcee (pas de doublon)
- consolidation_count incremente correctement

### v0.4e — Memory Linking (1-2 jours)

- [ ] `memory/consolidation.rs` : memory_linking()
- [ ] Scanner les entites co-creees dans une fenetre de 6h
- [ ] Creer un edge "temporally_linked" entre chaque paire
- [ ] Eviter les doublons : si le lien existe deja, renforcer la confidence
- [ ] Le lien temporel est bidirectionnel (A→B et B→A)

**Tests :**
- Entites creees a t et t+1h → lien temporel cree
- Entites creees a t et t+7h → pas de lien (hors fenetre)
- Lien deja existant → confidence augmentee
- N entites dans la fenetre → N*(N-1)/2 liens (combinatoire)

### v0.4f — Dream Cycle complet (1-2 jours)

- [ ] `dream_cycle()` : orchestrer les 6 etapes dans l'ordre
- [ ] 1. SHY downscaling → 2. Replay → 3. CLS transfer → 4. Memory linking → 5. Dark check → 6. GC optionnel
- [ ] Logging des stats du cycle (entites traitees, faits crees, dark nodes)
- [ ] Idempotent : relancer ne cree pas de doublons
- [ ] Configurable : chaque etape peut etre desactivee

**Tests :**
- dream_cycle() execute les 6 etapes dans l'ordre
- Stats retournees coherentes
- Idempotence : 2 appels consecutifs → pas de doublons
- Desactiver une etape : les autres s'executent quand meme

**Benchmark :** dream_cycle complet < 100ms sur 10K entities, < 1s sur 100K
