# Phase v0.2 — Perception (Dentate Gyrus + Cortex Prefrontal)

> Vector store SIMD, BM25, hybrid search RRF, deduplication.

---

## Sous-phases

### v0.2a — Vector Store SIMD (2-3 jours)
- [ ] `search/vector.rs` : stockage aligne, brute-force scan
- [ ] Cosine similarity AVX2 (x86_64)
- [ ] Cosine similarity NEON (AArch64)
- [ ] Fallback scalaire
- [ ] Dispatch runtime via OnceLock
- [ ] Pre-normalisation optionnelle (dot product si normalized)
- [ ] `add_entity()` accepte `Option<&[f32]>` embedding
- [ ] `vector_search(embedding, k)` retourne top-k

**Tests :**
- Cosine AVX2 vs scalar : memes resultats (tolerance 1e-5)
- vector_search retourne exactement k resultats
- Dimension mismatch → erreur

**Benchmark :** Cosine top-100 sur 100K vecs (384d) < 5ms

### v0.2b — BM25 Inverted Index (2-3 jours)
- [ ] `search/bm25.rs` : tokenizer, VByte posting lists
- [ ] Insert : tokenize + append posting + update doc_freq
- [ ] Search : decode postings + BM25+ scoring
- [ ] IDF lazy recomputation
- [ ] `text_search(query, k)` retourne top-k

**Tests :**
- "authentication" trouve les entites avec ce mot
- BM25 score respecte TF-IDF (plus d'occurrences = score plus haut)
- Entite sans texte : invisible au BM25 mais pas au vector search

**Benchmark :** BM25 search < 2ms sur 100K docs

### v0.2c — Hybrid Search RRF (1-2 jours)
- [ ] `search/hybrid.rs` : RRF fusion
- [ ] `search()` : combine vector + BM25 + activation boost
- [ ] Filtrage dark nodes (pour le futur)
- [ ] Options : top_k, include_dark, at_time

**Tests :**
- RRF retourne des docs trouves par les 2 legs en premier
- Entite trouvee par 1 seule leg : score plus bas
- Mode text-only (embedding_dims=0) fonctionne

### v0.2d — Deduplication triple (1-2 jours)
- [ ] `core/dedup.rs` : nom exact, Jaccard, cosine
- [ ] Check a l'insertion : si doublon detecte → retourne l'existant
- [ ] Seuils configurables

**Tests :**
- "hora-engine" et "Hora Engine" detectes comme doublons (normalisation)
- Embeddings similaires (cosine > 0.92) → dedup
- Noms differents mais meme embedding → dedup
